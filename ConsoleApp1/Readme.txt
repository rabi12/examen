Entregable

1. Codigo.
2. Se sigue principios SOLID. 
3. Se aplica inyección de dependencias en la fecha y en otras partes, sobretodo en constructores.
4. Bridge (No se distingue  ya que no fue necesario crear jerarquias, pero podria crearse) 
y Adapter (En el uso del JSON)
5. Se realizan algunas pruebas unitarias.
6. Se pueden agregar mas rangos de tiempo, por ejemplo semanas, quincenas, bimestres, etc,
	ya que bastaria con agregar la clase y configurar el metodo correspondiente.
   Se puede leer la configuracion en otro formato, xml, usando el mismo patron de adapter para configurarlo.
